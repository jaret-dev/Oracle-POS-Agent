# Reauthorizing Released Credit Card Authorizations

[Previous](t_payment_cash_back.htm) [Next](t_payment_one_credit_card.htm) JavaScript must be enabled to correctly display this content

## Reauthorizing Released Credit Card Authorizations

You can now reauthorize a previously released credit card authorization on the same check without having to re-swipe a customers card. You need to have the appropriate privileges to perform the reauthorization of a released credit card authorization.

1.  Reopen a closed check. For more information on how to reopen a closed check, see [Reopening a Closed Check](https://docs.oracle.com/en/industries/food-beverage/simphony/19.5/sipou/t_closed_check_reopen.htm#task-1-F731F6E3).
2.  In the check, highlight the payment.
3.  Select Void.
4.  In the check, highlight the authorization that needs to be reauthorized.
5.  Press the function key Reauthorize released CC Authorization. If successful, a new authorization appears.
    
    The following scenarios may also occur:
    
    -   If a user does not have the appropriate privileges to authorize, a prompt appears asking for a privileged user to authorize the transaction.
        
    -   If the Payment Service Provider (PSP) supports reauthorization but rejects the request, the user must obtain the card to process the authorization.
        
    -   If the PSP does not support reauthorization, a rejection message appears.
