# Using the Popup Condiment Orderer

[Previous](c_menu_items_condiments.htm) [Next](t_menu_items_declare_waste.htm) JavaScript must be enabled to correctly display this content

## Using the Popup Condiment Orderer

1.  Sign on to the workstation and begin a new check.
2.  Select a menu item which has required, or has optionally allowed condiment groups assigned to it. The Popup Condiment Orderer appears.
3.  Select a required condiment. The Popup Condiment Orderer automatically advances to the second group, and so on until you have selected the required number of condiments from all groups.
4.  If your location has set the condiment menu items to have a default count, or have the condiments set to match the parent menu items’ count, after the minimum count's threshold had been reached, click Done to move to the next condiment or to close the Popup Condiment Orderer.
    
    If you click Done without selecting the minimum count of a condiment group, the system remains on the current condiment group and the Done button text changes from Done to whatever is needed (for example, Select 2 to 3 Steak Sauce Current Count:1). When you use devices running the Android mobile operating system, you receive an error stating that the required condiment group is not complete for the parent menu item.
    
5.  To modify a previously ordered required or allowed condiment:
    1.  Select a menu item (condiment or parent) in the check detail area. The Condiment Edit Popup appears if it is configured from the Enterprise Management Console (EMC) RVC Parameters or Menu Item Classes modules.
        
        To edit a menu item when the Condiment Edit Popup is not configured:
        
        1.  Select the menu item (condiment or parent) in the check detail area.
            
        2.  Press the Recall Popup Condiment Orderer function key.
            
        
    2.  Change the condiments, and then press Done.
